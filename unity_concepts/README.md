<h2>Learning Objectives</h2>

<p>At the end of this project, you are expected to be able to <a href="/rltoken/A3aWeCPB_bUVZ2H0h0wq4Q" title="explain to anyone" target="_blank">explain to anyone</a>, <strong>without the help of Google</strong>:</p>

<ul>
<li>What is a GameObject</li>
<li>What is a Component</li>
<li>What is a Prefab</li>
<li>What is a Tag</li>
<li>What is a Layer</li>
<li>How to create and change the properties of a GameObject</li>
<li>How to create a Prefab</li>
<li>How to add Tags and Layers</li>
<li>Why is it important to name GameObjects and assets clearly and organize your project hierarchy</li>
<li>What are gameplay and game mechanics and how do they relate to developing any kind of interactive experience</li>
</ul>

<h2>Requirements</h2>

<h3>General</h3>

<ul>
<li>A <code>README.md</code> file, at the root of the folder of the project</li>
<li>Use Unity’s default <code>.gitignore</code> in your <code>holbertonschool-unity</code> directory – you may need to edit it so that it applies to your entire repo</li>
<li>Push the entire project folder <code>unity_concepts</code> to your repo, including <code>.meta</code> files – make sure that any files/folders in the <code>.gitignore</code> are not being pushed</li>
<li>Scenes and project assets such as Materials must be named and organized as described in the tasks</li>
<li>No scripts are allowed for this project</li>
</ul>
